사용자 채널을 작동시키는 인센티브 설계

사용자 채널(체크인·자가인증·신고)이 동기 부족으로 비활성화되는 위험을 차단하기 위한 인센티브·반대 인센티브. 모든 항목은 정책 협의 필요 사항으로 명시.

근로자 인센티브 (참여 동기)

  • 정책
    체크인 6개월 누적 → 정책자금·구직 가산점
    국민내일배움카드·고용보험 구직급여 처리 우선순위 가산
  • 즉시
    실시간 사업장 위험도 알림 (워치리스트)
    본인 사업장에 신호 누적 시 자동 통지
  • 즉시
    진정서 자동 생성·체불액 자동 계산
    신고 시간 단축 → 보복 노출 시간 ↓
  • 정책
    보복 발생 시 1350 우선처리 패스
    근로기준법 §104 위반 신고 자동 격상

사업주 인센티브 (자가인증 동기)

  • 정책
    정직 인증 사업장 → 정부 입찰 가산점
    조달청·공공기관 입찰 평가 시 가점
  • 정책
    정직 인증 → 정책자금 우대 금리
    근로복지공단·신용보증기금 우대
  • 시장
    구직자에 노출 — 우수 인력 확보 경쟁력
    워크넷·잡코리아 등 채용 플랫폼에 인증 표시 협의
  • 반대
    자가인증 + 근로자 체크인 불일치 → 자동 우선 점검
    "지급함" 보고했는데 근로자 다수 미수령 → 거짓 신고로 처분 가중

근로감독관 인센티브

  • • 우선순위 큐 자동 정렬로 점검 시간 30%+ 절감 (벤치마킹 — 호주 Fair Work)
  • • 페이퍼컴퍼니 그래프로 동일 대표자 신규 사업체 추적 자동화
  • • 도메인 분리 격상 규칙 → false positive 감소 → 인력 부담 ↓

📅 4단계 점진적 도입 로드맵

Phase 0 — PoC 시범 (0~3개월)
  • 특정 산업·지역 1곳 시범 (예: 제주 숙박업 / 서울 학원업)
  • 근로자 1,000명 / 사업주 100명 자발적 참여
  • 출입국·1350 행정 데이터 미연계 — 사용자 채널만
Phase 1 — SDK 한 줄 이식 (3~12개월)
  • 고용24·정부24 신청 페이지 SDK 한 줄 추가 (M6 RDP 탐지)
  • 워크넷 채용공고에 정직 인증 마크 노출 협의
  • 전국 자발적 등록 사업주 1만 명 확보 목표
Phase 2 — 행정 데이터 연계 (12~24개월)
  • 법무부 출입국 매칭 행정정보 공동이용 협의 → M6 Phase 2 가동 (F1 0.99)
  • 4대보험 통합 정보 / 국세청 휴폐업 시계열 연계
  • 1350 진정 처리 결과 자동 회신 (폐쇄루프)
Phase 3 — 제도 정착 (24개월+)
  • 정직 인증을 공공입찰·정책자금 평가에 정식 반영 (시행규칙 개정 협의)
  • ML 모델 정식 도입 (라벨 누적 후)
  • 전국 사업장 50%+ 자발적 인증 목표

⚠ 현재 시스템 한계

  • • 실배포 A/B 데이터 부재 — Phase 1 도입 후 6개월이면 확보 가능
  • • 사용자 채널 의존성 — 인센티브 정책 협의 없으면 데이터 0 위험
  • • L5 출입국 매칭은 시뮬만 — 정책 협의 사안 (수년 소요)
  • • ML 표본 부족 — 789건 라벨로는 룰베이스가 한계. ML은 v2
  • • SDK 정부 보안심사 — 코드 양도 + 정부 환경 호스팅으로 우회. 입찰·계약 절차 별도
