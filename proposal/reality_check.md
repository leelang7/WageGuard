## 1. 카테고리별 정직한 평가

### A. 유지 (실데이터·실효성 명확) ★★★

| 모듈 | 근거 |
|---|---|
| 체불사업주 명단 789건 EDA | 공식 라벨, 분석 가치 명확 |
| 국세청 사업자상태(NTS) | 실 API, 휴/폐업 객관 |
| EIS 고용행정통계 | 실 API, 거시 컨텍스트 |
| 사업장 통합 프로필 `/company/{name}` | 다중 데이터 결합 |
| **체불액 자동 계산기** `/wage-calc` | 근로기준법 명확, 실용 |
| **명세서 자동 검사** `/payslip-check` | §48 룰 명확 |
| **신고+증빙+PDF 진정서** `/cases` | 진짜 행정 양식 |

→ 본 출품작의 **valid claim** 은 이 7개에서 옴. 나머지는 보조.

### B. 단순화·통합 (가치는 있지만 운영 부담)

| 결정 | 이유 |
|---|---|
| 사업주 페이지 3개 (`/owner` + `/attest` + `/owner-notice`) → **통합 진입** | 사용자가 사업주 도구를 한 곳에서 시작 |
| 근로자 진입을 `/cases` 중심으로 통합 (체크인·워치는 보조 링크) | 신고가 가장 강한 채널 |
| `/m6/embed-demo` SDK 데모 → 보조 | 정부 도입은 정책 사안 |
| `/demo` 자동 시연 → 우측 보조 | 발표용. 실서비스 X |

### C. DROP (실효성 의문 / 키·정책 미해결)

| 모듈 | 결정 | 이유 |
|---|---|---|
| 부정당업자 명단 (`/api/blacklist`) | 메인 흐름에서 **제외** | CSV·키 없으면 0건 |
| 국민연금 탈퇴사업장 | **제외** | activation propagation 안 됨 |
| OpenDART 자리 | **제외** | 키 없으면 0건 |

코드는 유지(향후 재개 가능)하되 메인 nav에서 노출 X.

### D. 정당성 보강 (통계적 명시)

| 모듈 | 보강 |
|---|---|
| 위험점수 산출 | `risk_confidence` (high/medium/low) + `sample_note` 명시 |
| 페이퍼컴퍼니 그래프 | `disclaimer` "동일 주소 ≠ 페이퍼" — 영세 빌딩 false positive 경고 |
| 셀(업종×지역) 점수 | "명단 분포 기반 — 모집단 편향 가능" 명시 |
| 모든 응답 | `risk_disclaimer` "보조 지표, 행정 처분 근거 X" 자동 첨부 |

## 2. 시스템 정체성 — 솔직한 재정의

### 우리가 진짜로 만든 것

> **임금체불 신고 진입장벽 제거 + 다중 데이터 결합 시각화 + 사용자 채널 인프라**

### 우리가 만들지 못한 것

| 못 하는 것 | 이유 |
|---|---|
| 신호 없는 회사 자동 사전 탐지 | 행정 비공개 데이터 접근 불가 |
| 부정수급 차단 (실배포) | L5 출입국 매칭 정책 협의 미해결 |
| 공식 행정 처분 | 정부 영역 — 우리는 도구 |

### 그래도 가치 있는 것

1. **신고 진입장벽 제거**: 체불액 자동 계산 + 진정서 자동 생성 + 증빙 첨부 UI = 신고 시간 ↓ → 보복 노출 시간 ↓
2. **다중 데이터 결합**: NTS·EIS·체불명단·검색·NPS를 사업장 1개 키로 통합 조회
3. **사용자 채널 인프라**: 매월 체크인·신고+증빙·자가인증 — 사용자 참여 시 사전 탐지 가능 (가설)

|---|---|---|
