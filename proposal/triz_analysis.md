## 가장 강력한 TRIZ 도출 아이디어 (즉시 박음)

### A. 정직 사업주 자가 인증 (Principle #13 + #25)

> **사업주가 매월 명세서·임금지급 사실을 자가 등록 → 시스템이 일관성 검증 → 누적 통과 시 "WageGuard 정직 인증" 디지털 마크 발급.**

- 채용공고·구직자에 노출 → **시장 압력으로 정화**
- 단방향 적발이 아닌 **양방향 인센티브**
- 정직한 사업주가 자기 자격을 자동 입증할 도구가 됨
- 미인증 사업장은 자동으로 "정직성 미입증" 잠재 위험

### B. 집단 보호 알림 (Principle #22)

> **신고 발생 시 같은 업종·지역의 워치리스트 등록자에 익명 통계 알림.**

"당신 사업장 인근 OO업종에서 체불 신고 N건 누적" → 본인 사업장도 점검·증거 수집 시작 유도. 신고가 다른 신고를 부르는 **선순환**.

### C. 임금 약속 공시 (Principle #24)

> **사업주가 임금 지급일을 시스템에 공시 → 그 날 ±N일 NPS·NTS 변동 자동 모니터링 → 위반 자동 신호.**

사업주의 자발적 약속이 시스템의 자동 검증 매개체가 됨.

## 기존 시스템의 모순 해소 — TRIZ 결과

| 모순 | TRIZ 해결 |
|---|---|
| 적발↑ vs 정직 사업주 부담↑ | **자가인증** (정직 사업주는 자동 면제 + 마크) |
| 모니터링↑ vs 프라이버시↑ | **공시·자가신고** (사업주가 스스로 데이터 제공) |
| 신고 접근성↑ vs 진위↑ | **다중 사용자 신뢰도** (집단지성) |
| 적발↑ vs 행정 부담↑ | **시장 압력** (인증 마크 → 자정작용) |
