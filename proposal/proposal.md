# PayRadar — 두 가지를 한 플랫폼에

> ① 근로자용 자동 임금 안전 알림 (B2C 무료) + ② 부정수급 차단 SDK (B2G 정부 라이선스)
> 제5회 고용노동 공공데이터·AI 활용 공모전 출품작
> 주관기관: **한국고용정보원·한국장애인고용공단** / 마감: 2026.05.14

---

## 1. 한 줄 정의

**근로자에게는 무료 자동 임금 알림 도구**, **정부에는 한 줄 이식 부정수급 차단 SDK** — 두 가지를 한 플랫폼으로 동시에 제공.

---

## 2. 컨셉 재구성 — 왜 두 코어로 압축했나

기존 시스템 대비 본 출품작이 추가하는 가치:
- 운영자(감독관) 중심 → 근로자 직접 가치 없음
- 신호 없는 회사 못 잡음
- 부정수급은 IP 단일 검증으로 끝 → 우회 쉬움

본 출품작:
- **근로자에게 직접적 가치** (코어 A) — 매월 자동 알림으로 인지 격차 해소
- **정부에게 즉시 효과** (코어 B) — 한 줄 SDK로 보안심사 친화적 통합

---

## 3. 코어 A — 자동 임금 안전 알림 (근로자 B2C 무료)

### 문제
- 근로자가 임금체불·부정 행위를 즉각 인지할 수 있도록 자동 신호화 필요
- 신고·진정 절차 진입장벽 큼 (양식·법률·증빙 부담)
- 영세 사업장 근로자는 정보 비대칭이 더 심함

### 해결
1. **회사명 1번 입력** → 임금 안전 점수 + 같은 사업장 동료 보고 + 행동 가이드
2. **매월 자동 미수령 탐지** — 임금일 +N일 후 1탭 체크인
3. **체불 발생 D+0~D+90 행동 타임라인** — 시점별 해야 할 일
4. **체불액 자동 계산기 + 진정서 PDF 자동 생성** — 신고 시간 ↓ → 보복 노출 시간 ↓
5. **사업주 자가인증과 자동 대조** — 거짓 자가인증 자동 탐지

### 차별화
- 신호 없는 회사도 근로자 1명만 등록하면 시스템 진입 (사용자가 데이터를 만든다)
- 모바일 한 화면 진입 (`/me`, `/kakao` 봇 시뮬)
- 익명 결과 카드 (사업장명 비공개·점수만)

### 수익 모델
- **사용자 완전 무료**
- 정부 위탁 운영 (고용노동부) 또는 비영리 재단 후원
- 정직 인증 사업장에 채용공고 노출 권 (선택적 광고 모델)

### 연 사용자 가치 추정
- 임금체불 발생 36만 명 / 1.7조 원
- 도구 활용 시 신고 격상 효과 +30% (Fair Work 벤치마킹)
- → 연 사회적 가치 **5,400~10,000억** (몬테카를로 95% CI)

## 4. 코어 B — 부정수급 차단 SDK (정부 B2G 라이선스)

### 문제 — 현장 제보 우회 사례
한 시민이 해외 체류 중 한국 본가 PC에 원격 접속해 실업급여 신청을 통과시킨 사례가 익명 제보로 확인되었다. IP는 한국으로 인식되고 본인 인증서·SMS도 정상 통과 — **현행 시스템은 이를 사전에 거를 방법이 없다**.

### 해결 — 9 신호 + 한 줄 SDK
- L1 IP 국가 (단순) / L2 환경 (timezone·언어·WebRTC 누출) / L3 행동 (마우스 jitter·키 입력 분산) / L4 디바이스 (WebGL 가상 어댑터) / L5 행정 매칭 (출입국·4대보험·로밍 — 정책)
- 통합 방식: `<script src="payradar-sdk.js">` 한 줄

### 차별화
- **현장 제보 기반** — 익명 제보로 확인된 실제 우회 패턴을 직접 잡는 시스템 설계
- **외부 데이터 유출 0** — 정부 환경 자체 호스팅. 보안심사 친화
- **출입국 매칭 결합 시 F1 0.83 → 0.995**

### 수익 모델
- **정부 기관 연간 라이선스** (고용보험공단·근로복지공단·공단 단위)
- 코드 양도 + 정부 환경 호스팅 = SaaS 운영 X (분쟁·보안 책임 ↓)
- 알고리즘·룰 자문 형태 — 보안심사 통과 정공법

### 연 절감 효과 추정
- 부정수급 환수 +5%p × 부정수급 비중 = 연 ~850억
- 신청 시간·인력 효율화 별도

## 6. 기술 검증 (출품 시점)

| 검증 | 수치 |
|---|---|
| **ML 정식 모델** Logistic Regression (pure Python) | 학습 정확도 0.78, K-fold 5 CV 0.735±0.04, F1 0.715 |
| K-fold (5) — 업종/지역 priors | 업종 0.81±0.03, 지역 0.73±0.03 |
| M6 시뮬 1,000건 | 기본 F1 0.83 / 출입국 연계 F1 0.995 |
| Ablation (도메인 분리 효과) | FP 0건 유지 + Recall 보존 |
| 몬테카를로 1,000회 | 95% CI: 2,060~9,968억/년 |
| 명세서 검사 | §48 위반·최저임금·4대보험 누락 자동 검출 |
| 체불액 계산 | 한국 근로기준법 §50/53/55/56 + 퇴직금 |

### AI 활용 명시 (공모전 필수 요건 충족)
- 룰베이스가 아닌 **학습된 가중치** 모델 — `/api/ml/info`, `/api/ml/cv`, `/api/ml/predict` 라이브 엔드포인트
- 7개 특성 × Logistic Regression × 200 epoch gradient descent
- 가중치 해석 가능 (블랙박스 아님) — `/ml` 페이지에서 즉시 확인
